# Issues: Пробковая доска для TTRPG

---

## Issue 1: Project scaffold + Canvas Engine

**Type:** AFK
**Blocked by:** None — can start immediately

### What to build
Базовая структура проекта и бесконечный canvas с pan/zoom. Никаких карточек — только координатная система, обработка событий мыши/клавиатуры и viewport. Это фундамент для всех остальных модулей.

### Acceptance criteria
- [ ] SPA запускается в браузере без ошибок
- [ ] Canvas бесконечный, поддерживает pan через пробел+drag и зажатое колесико
- [ ] Zoom работает через колесико мыши
- [ ] Координатная система корректно переводит экранные координаты в мировые

### User stories addressed
- User story 10

## Issue 2: Card Module — создание и редактирование

**Type:** AFK
**Blocked by:** Issue 1

### What to build
Полный цикл работы с карточкой: создание двойным кликом, выбор размера перетаскиванием, свободный текст, перемещение ПКМ, z-order (карточка поверх карточки). Три стандартных размера карточки.

### Acceptance criteria
- [ ] Двойной клик на пустом месте создаёт карточку, сразу можно печатать
- [ ] Клик+drag на пустом месте рисует размер новой карточки
- [ ] Доступны 3 фиксированных размера карточки
- [ ] ПКМ+drag перемещает карточку по canvas
- [ ] Карточки можно накладывать друг на друга (z-order)
- [ ] Одиночный клик по карточке открывает редактирование текста

### User stories addressed
- User stories 1, 2, 5, 6, 8, 9

## Issue 3: Card types — визуальные маркеры

**Type:** AFK
**Blocked by:** Issue 2

### What to build
Пять типов карточек (NPC, Локация, Событие, Идея, Generic). Тип обозначается формой срезанного уголка карточки. Цвет уголка/среза не зависит от слоя — только форма. Тип выбирается при создании или меняется после.

### Acceptance criteria
- [ ] При создании карточки предлагается выбор типа
- [ ] Тип можно изменить на уже существующей карточке
- [ ] Каждый тип имеет уникальный визуальный маркер (форма/размер срезанного уголка)
- [ ] Маркер типа не конфликтует с цветом слоя

### User stories addressed
- User stories 3, 4

## Issue 4: Thread Module — нитки

**Type:** AFK
**Blocked by:** Issue 2

### What to build
Нитки между карточками. Левый клик по краю карточки начинает нитку, drag до другой карточки завершает. Булавка создаётся автоматически при первом подключении нитки к карточке. Нитка визуально провисает (cubic bezier). Реализация через SVG overlay поверх canvas.

### Acceptance criteria
- [ ] Левый клик по краю карточки + drag начинает нитку
- [ ] Отпускание над другой карточкой завершает нитку
- [ ] Булавка автоматически появляется на карточке при первом подключении нитки
- [ ] Последующие нитки привязываются к той же булавке
- [ ] Нитка отображается как провисающая кривая (не прямая линия)
- [ ] Нитки корректно перемещаются вместе с карточками

### User stories addressed
- User stories 11, 12, 13

## Issue 6: Layer Manager — два слоя

**Type:** AFK
**Blocked by:** Issues 2, 4

### What to build
Два переключаемых слоя: лорный и логический. Карточка может существовать в одном или обоих слоях и содержать разный текст в каждом. Нитки принадлежат конкретному слою. При переключении слоя мгновенно меняются фон, цвет карточек и цвет ниток. Неактивный слой можно приглушить или скрыть полностью — настраиваемый параметр.

### Acceptance criteria
- [ ] Кнопка/хоткей переключает активный слой мгновенно
- [ ] Карточка может быть назначена в один или оба слоя
- [ ] Карточка в двух слоях содержит независимый текст в каждом
- [ ] При переключении слоя карточка остаётся на том же месте
- [ ] Цвет карточек и ниток определяется активным слоем автоматически
- [ ] Фон доски меняется при переключении слоя
- [ ] Неактивный слой можно настроить: от полупрозрачности до полного скрытия
- [ ] Нитка принадлежит конкретному слою и видна только в нём (или приглушена)

### User stories addressed
- User stories 15, 16, 17, 18, 19, 20, 21

## Issue 7: Background Renderer + Minimap

**Type:** AFK
**Blocked by:** Issue 1

### What to build
Визуальная среда доски: пробковая текстура в рабочей зоне с виньетированием к краям (мягкие границы без жёсткого обрезания). Миникарта в углу экрана — масштабированный вид всей доски с индикатором текущего viewport, клик для навигации.

### Acceptance criteria
- [ ] Фон имеет пробковую текстуру в центральной зоне
- [ ] К краям canvas фон постепенно светлеет/выцветает (нет жёсткой границы)
- [ ] Миникарта отображается в углу и показывает все карточки
- [ ] Индикатор на миникарте отражает текущую позицию viewport
- [ ] Клик на миникарту перемещает viewport в соответствующую точку

### User stories addressed
- User stories 23, 24

## Issue 8: Gravity System + Shift+колесико

**Type:** AFK
**Blocked by:** Issue 2

### What to build
Два небольших поведения карточек. Гравитация: новые карточки создаются ближе к центроиду существующих (мягко, не принудительно). Shift+колесико над карточкой меняет размер текста на ней.

### Acceptance criteria
- [ ] Новая карточка появляется со смещением к центру существующих карточек
- [ ] Гравитация мягкая — не мешает создавать карточки в произвольном месте
- [ ] Shift+колесико над карточкой увеличивает/уменьшает размер текста
- [ ] Размер текста сохраняется в состоянии карточки

### User stories addressed
- User stories 7, 25

## Issue 10: Persistence Layer

**Type:** AFK
**Blocked by:** Issue 2

### What to build
Автосохранение всего состояния доски в localStorage при каждом изменении (с дебаунсом). Загрузка состояния при старте приложения. Пустой localStorage не ломает приложение.

### Acceptance criteria
- [ ] Состояние доски автоматически сохраняется в localStorage после каждого изменения
- [ ] Дебаунс предотвращает избыточные записи при быстрых изменениях
- [ ] При открытии приложения состояние восстанавливается из localStorage
- [ ] Приложение корректно стартует с пустым localStorage (первый запуск)

## Issue 11: Export Module — Obsidian + JSON

**Type:** AFK
**Blocked by:** Issues 6, 10

### What to build
Два вида экспорта. Obsidian: каждая карточка → отдельный `.md` файл, нитки → `[[wikilinks]]`, слой → теги (`#лор` / `#мета`), тип → тег. JSON: полный экспорт/импорт состояния доски для резервных копий и переноса между устройствами.

### Acceptance criteria
- [ ] Кнопка экспорта генерирует папку `.md` файлов
- [ ] Каждая карточка экспортируется в отдельный файл, имя = текст карточки (или первая строка)
- [ ] Нитки экспортируются как `[[wikilinks]]` внутри файла карточки
- [ ] Слой карточки экспортируется как тег (`#лор`, `#мета` или оба)
- [ ] Тип карточки экспортируется как тег (`#npc`, `#локация` и т.д.)
- [ ] Карточки с двумя слоями имеют оба раздела в `.md` файле
- [ ] JSON-экспорт сохраняет полное состояние доски
- [ ] JSON-импорт восстанавливает состояние (round-trip без потерь)

### User stories addressed
- User stories 26, 27, 28, 29, 32

## Issue 12: Планшетный режим

**Type:** AFK
**Blocked by:** Issues 2, 4, 6

### What to build
Режим просмотра для планшета: pan и zoom без редактирования. Адаптивный viewport. Все интерактивные элементы редактирования скрыты или задизейблены.

### Acceptance criteria
- [ ] На планшете доска открывается в режиме только для чтения
- [ ] Pan и zoom работают через тач-жесты
- [ ] Редактирование карточек, создание ниток и других объектов недоступно
- [ ] Viewport адаптируется к размеру экрана планшета
- [ ] Все карточки, нитки и изображения корректно отображаются
