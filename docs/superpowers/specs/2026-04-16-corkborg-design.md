# Design: Corkborg — Пробковая доска для TTRPG

**Date:** 2026-04-16  
**Status:** Approved

---

## Problem

Мастер TTRPG хочет визуально планировать сюжеты: создавать записки, размещать их на доске и натягивать нитки между ними. Существующие инструменты (Miro, FigJam) перегружены, имеют корпоративное ощущение и реализуют связи как диаграммные стрелки, а не нитки. Нужен минималистичный инструмент, имитирующий физическую доску, с поддержкой двух смысловых слоёв и экспортом в Obsidian.

---

## Solution

SPA на **Svelte + Vite + TypeScript**, без бэкенда. Бесконечный canvas с тактильной эстетикой пробки, карточками 5 типов, провисающими нитками через SVG overlay, двумя переключаемыми слоями и экспортом в Markdown/JSON. Данные хранятся в `localStorage`.

---

## Architecture

```
src/
  stores/          # Svelte stores — единый источник истины
    board.ts       # Состояние доски: карточки, нитки, изображения
    layer.ts       # Активный слой, степень приглушения
    viewport.ts    # Pan/zoom, трансформация координат
  components/
    Canvas.svelte        # Корневой компонент, обработка событий мыши
    Card.svelte          # Карточка с типом, слоем, текстом
    ThreadLayer.svelte   # SVG overlay со всеми нитками
    Thread.svelte        # Одна нитка (cubic bezier)
    Pin.svelte           # Булавка на карточке
    Minimap.svelte       # Миникарта в углу
    Background.svelte    # Пробковая текстура + виньетирование
    ImageNode.svelte     # Изображение на canvas
    LayerSwitch.svelte   # Кнопка переключения слоёв
  lib/
    geometry.ts    # Координатные вычисления (screen ↔ world)
    bezier.ts      # Вычисление провисающей кривой
    gravity.ts     # Центроид существующих карточек
    export.ts      # Генерация md-файлов и JSON
    persistence.ts # localStorage с дебаунсом
  types.ts         # Card, Thread, Pin, Image, Layer, BoardState
```

**Принцип:** каждый модуль имеет одну ответственность и общается через Svelte stores. Canvas.svelte не знает про экспорт; export.ts не знает про рендеринг.

## Data Model

```typescript
type LayerKey = 'lor' | 'meta'

type Card = {
  id: string
  type: 'npc' | 'location' | 'event' | 'idea' | 'generic'
  x: number; y: number          // мировые координаты
  width: number; height: number // один из 3 стандартных размеров
  content: Record<LayerKey, string>
  layers: LayerKey[]            // ['lor'] | ['meta'] | ['lor', 'meta']
  zIndex: number
  fontSize: number
  pinId: string | null          // id булавки (создаётся при первой нитке)
}

type Thread = {
  id: string
  layer: LayerKey
  from: { cardId: string }
  to: { cardId: string }
  label: string | null
}

type Pin = {
  id: string
  cardId: string
  // позиция вычисляется из карточки, не хранится отдельно
}

type ImageNode = {
  id: string
  x: number; y: number
  width: number; height: number
  dataUrl: string               // base64, хранится в localStorage
}

type BoardState = {
  cards: Card[]
  threads: Thread[]
  pins: Pin[]
  images: ImageNode[]
}
```

## Modules

### Canvas Engine
Бесконечный canvas реализован через CSS `transform: translate + scale` на контейнере. Координатная система: все объекты хранят **мировые** координаты, `geometry.ts` переводит экранные ↔ мировые при каждом событии мыши.

**Pan:** пробел+drag или зажатое колесико — обновляет `viewport.ts`.  
**Zoom:** колесико — масштаб относительно курсора.  
**Не используем `<canvas>`-элемент** — всё DOM/SVG, что упрощает события и текстовые поля.

### Card Module
Двойной клик на пустом месте — создаёт карточку в мировых координатах курсора (с поправкой на гравитацию). ПКМ+drag — перемещает. Три фиксированных размера: S (120×80), M (180×120), L (240×160) px в мировых единицах.

**Тип карточки** — форма срезанного уголка (clip-path или border-radius на уголке): разный угол и размер среза для каждого типа. Цвет среза не используется — цвет занят слоем.

**z-order:** `zIndex` в состоянии, при клике на карточку она поднимается на вершину стека.

### Thread Module
SVG overlay (`position: absolute`, `pointer-events: none` кроме самих ниток). При hover по краю карточки появляется зона для начала нитки (cursor изменяется). Drag от края до другой карточки создаёт Thread.

**Форма:** cubic bezier. Контрольные точки смещены вниз от середины отрезка — имитируют провисание. Формула: `cp1 = midpoint + (0, sag)`, `cp2 = midpoint + (0, sag)`, где `sag = distance * 0.3`.

**Булавка:** создаётся автоматически при первой нитке карточки. Все последующие нитки той же карточки крепятся к той же булавке. Булавка рендерится как маленький кружок поверх карточки.

### Layer Manager
Svelte store `layer.ts` хранит `activeLayer: LayerKey` и `ghostOpacity: number (0–1)`. CSS-переменные (`--layer-bg`, `--card-color`, `--thread-color`) обновляются реактивно через `$:` в корневом компоненте. Переключение мгновенное.

Карточки и нитки не в активном слое рендерятся с `opacity: ghostOpacity`. При `ghostOpacity = 0` — полностью скрыты.

### Background Renderer
CSS radial-gradient поверх SVG-паттерна пробковой текстуры. Паттерн — inline SVG с шумом (feTurbulence). Виньетирование — radial-gradient от прозрачного в центре к светло-бежевому к краям. Фон меняет цвет при переключении слоя через CSS-переменную.

### Minimap
Фиксированный `<div>` в правом нижнем углу. Рендерит упрощённые прямоугольники всех карточек в масштабе 1:20 (примерно). Прямоугольник viewport-индикатора вычисляется из `viewport.ts`. Клик на миникарту — `viewport.pan` до соответствующей мировой точки.

### Gravity System
При создании карточки `gravity.ts` вычисляет centroid всех существующих карточек. Новая карточка смещается на 30% расстояния к centroid от позиции курсора. Если карточек нет — позиция курсора без смещения.

### Export Module
**Obsidian экспорт:** для каждой карточки генерирует `.md` файл. Имя файла = первая строка контента (sanitized). Wikilinks берутся из Thread-ов. Теги из `layers` и `type`. Использует File System Access API (`showDirectoryPicker`) — пользователь выбирает папку.

**JSON экспорт/импорт:** `JSON.stringify(boardState)` → файл. Импорт через `<input type="file">`.

### Persistence Layer
`persistence.ts` подписывается на `board` store через Svelte `$board`. При каждом изменении — дебаунс 500ms → `localStorage.setItem('corkborg-board', JSON.stringify(state))`. При старте — `getItem` + JSON.parse, fallback на пустое состояние.

## Controls

| Действие | Управление |
|---|---|
| Создать карточку | Двойной клик на пустом месте |
| Нарисовать размер карточки | Клик + drag на пустом месте |
| Переместить карточку | ПКМ + drag |
| Переместить viewport | Пробел + drag, зажатое колесико |
| Начать нитку | Левый клик по краю карточки + drag до другой |
| Изменить размер текста | Shift + колесико над карточкой |
| Редактировать карточку | Одиночный клик |
| Переключить слой | Кнопка в UI или Tab |
| Добавить лейбл нитки | Одиночный клик по нитке |

## Visual Design

**Лорный слой:** тёплая палитра, бежево-пробковый фон, карточки кремового оттенка, нитки красно-коричневые.  
**Логический слой:** холодная палитра, серо-синий фон, карточки светло-голубые, нитки синие.  
**Тип карточки:** форма срезанного уголка (clip-path), независимо от слоя.

---

## Testing

- **Card Module:** создание с контентом по слоям, назначение в слои, независимость текста в слоях.
- **Thread Module:** создание нитки, автосоздание булавки, принадлежность слою.
- **Layer Manager:** переключение меняет активный слой, видимость карточек по слоям.
- **Export Module:** md-генерация, wikilinks из ниток, теги из слоёв, JSON round-trip.
- **Persistence:** сохранение/восстановление, корректный старт с пустым localStorage.

## Out of Scope

- Мультипользовательский режим
- Аккаунты и облачное хранилище
- Мобильное редактирование (только просмотр на планшете)
- Направленные стрелки
- Структурированные поля карточек
- Более двух слоёв
- Интеграция с Roll20/Foundry
- Undo/redo за пределами базового Ctrl+Z
